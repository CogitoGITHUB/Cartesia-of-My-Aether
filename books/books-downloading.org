* Overview

This file documents the workflow for downloading free public domain books
and converting them to org-mode format for Emacs.

* MANDATORY Workflow (ONE STEP AT A TIME)

ALWAYS follow these steps in order. NEVER skip a step.

** Step 1: SEARCH
- Use websearch to find books from MULTIPLE sources
- Check Gutenberg, Internet Archive, Standard Ebooks, Open Library
- NEVER use only one source

** Step 2: DOWNLOAD
- Get the direct URL for the book
- Verify language is English BEFORE downloading
- Use curl to download to .txt format

** Step 3: CONVERT
- Use pandoc to convert .txt to .org format
- Command: cat "[file].txt" | pandoc -f markdown -t org -o "[file].org"
- Delete the .txt file AFTER conversion

** Step 4: RENAME
- Rename the .org file with proper book title
- Format: Title-Author.org
- Example: The-Republic-Plato.org

** Step 5: VERIFY (CRITICAL)
- ALWAYS verify after each step completes
- Check: head -20 "[file].org" (should have real content, not HTML)
- Check: wc -l "[file].org" (should be thousands of lines, not <10)
- Check: tail -10 "[file].org" (should have end content)

IF ANY CHECK FAILS:
- Delete the file
- Go back to Step 1 (different source) or Step 2 (different URL)

** Step 2 (continued): Download from Project Gutenberg

Project Gutenberg provides direct file URLs for each book:

: https://gutenberg.org/files/[EBOOK_ID]/[EBOOK_ID].txt       (plain text)
: https://gutenberg.org/files/[EBOOK_ID]/[EBOOK_ID].html      (HTML)
: https://gutenberg.org/files/[EBOOK_ID]/[EBOOK_ID]-0.txt    (alternate text)
: https://www.gutenberg.org/ebooks/[EBOOK_ID].txt.utf-8    (UTF-8 text)

Download with curl:
: curl -sL "[URL]" -o "[output-filename]"

** Step 2b: Download from Internet Archive (archive.org)

Internet Archive has millions of texts not on Gutenberg. Use the =_djvu.txt= file:

: curl -sL "https://archive.org/download/[ITEM_ID]/[ITEM_ID]_djvu.txt" -o "output.txt"

Getting the ITEM_ID:
1. Search archive.org for the book
2. Note the ID from the URL: archive.org/details/[ITEM_ID]
3. Check available files via API:
: curl -s "https://archive.org/metadata/[ITEM_ID]" | grep -o '"name":"[^"]*"'

Common patterns:
: [ITEM_ID]_djvu.txt          (most reliable)
: [ITEM_ID]_text.txt          (sometimes available)
: [ITEM_ID].txt              (rare)
: [ITEM_ID].pdf             (fallback, not convertible)

Download example:
: curl -sL "https://archive.org/download/lecturesonjurisp11aust/lecturesonjurisp11aust_djvu.txt" -o "book.txt"

** Step 2c: Download from Standard Ebooks

Standard Ebooks has curated, well-formatted editions but requires different approach:

1. Search: https://standardebooks.org/ebooks
2. Use their OPDS/API or download via website
3. Extract text from epub:
: unzip -p "book.epub" *.xhtml | pandoc -f html -t org -o "book.org"

Alternative - download all via bulk:
: curl -sL "https://standardebooks.org/ebooks?format=epub" -o "catalog.xml"

** Step 2d: Download from Open Library

Open Library redirects to Internet Archive or other sources:

1. Find book at https://openlibrary.org
2. Check "Download options" section
3. Use curl with the direct URL:
: curl -sL "[DIRECT_URL]" -o "book.txt"

** Step 3: Inspect the text structure

Find chapter boundaries and structure:
: grep -n "CHAPTER\|PREAMBLE\|Part [A-Z]" "[file]"

Find aphorism/section numbering:
: grep -n "[0-9]\+\. " "[file]"

Find ebook metadata:
: grep -n "Title:\|Author:\|Release Date:" "[file]"

** Step 4: Convert plain text to org-mode

Use pandoc (preferred, supports org output):
: pandoc -f html -t org "[input.html]" -o "[output.org]"

Use pandoc (plain text):
: pandoc -f plain -t org "[input.txt]" -o "[output.org]"

Alternative: Node.js script for custom parsing.

* Step-by-Step Example: Beyond Good and Evil

** Step 1: Search

** Step 2: Find Gutenberg ID

** Step 3: Download plain text

** Step 4: Inspect structure

# 841: CHAPTER II. THE FREE SPIRIT
# 1456: CHAPTER III. THE RELIGIOUS MOOD

** Step 5: Extract body
Find start of main content (after header):
: grep -n "CHAPTER I. PREJUDICES" "[file]"

Find end marker:
: grep -n "END OF THIS PROJECT" "[file]"

** Step 6: Convert with Node.js
#+begin_src javascript
const fs = require('fs');

let data = fs.readFileSync('input.txt', 'utf8');

// Extract body between start and end markers
let start = data.indexOf('START_MARKER');
let end = data.lastIndexOf('END_MARKER');
data = data.substring(start, end);

const lines = data.split('\n');
let output = [];

// Add org-mode headers
output.push('#+TITLE: Book Title');
output.push('#+AUTHOR: Author Name');
output.push('');

// Parse chapters and content
lines.forEach((line) => {
  // Detect chapter headers
  const chapterMatch = line.match(/^CHAPTER\s+[IVX]+\.\s+(.+)$/);
  if (chapterMatch) {
    if (output.length) output.push('');
    output.push('* ' + chapterMatch[1].trim());
    output.push('');
    return;
  }
  // Buffer paragraphs, flush on blank lines
  if (line.trim().length < 3) {
    if (paragraphBuffer.length) {
      output.push(paragraphBuffer.join(' ').trim());
      paragraphBuffer = [];
    }
    return;
  }
  paragraphBuffer.push(line);
});

fs.writeFileSync('output.org', output.join('\n'));
#+end_src

: wc -l "output.org"
: ls -lh "output.org"

* Common Book Formats (All Sources)

| Source | Extension | Format    | Best For                          |
|--------|-----------|-----------|-----------------------------------|
| Gutenberg | .txt      | Plain text| Direct conversion, cleanest       |
| Gutenberg | .html     | HTML      | Pandoc conversion                 |
| Gutenberg | .epub     | EPUB      | E-readers, pandoc                 |
| Archive.org | .djvu.txt | OCR text | IA primary format           |
| Archive.org | .pdf      | PDF       | Fallback only (not convertible) |
| Archive.org | .hocr.html | HTML     | Sometimes available             |
| StdEbooks | .epub     | EPUB      | Best quality, requires unzip  |

** Source Priority

1. Project Gutenberg (=.txt=) — cleanest, most reliable
2. Internet Archive (=_djvu.txt=) — for books not on Gutenberg
3. Standard Ebooks (=.epub=) — best formatting, requires extraction
4. Open Library — redirects to other sources

* Tips

1. Always prefer multiple sources — don't just use Gutenberg
2. For texts, prefer plain text (=.txt=) over HTML — fewer artifacts
3. On Archive.org, always try =_djvu.txt= first (most reliable IA format)
4. Check both =[ebook-id].txt= and =[ebook-id]-0.txt= variants on Gutenberg
5. Strip Gutenberg/IA header/footer before converting
6. Pandoc with =--wrap=none= prevents line wrapping
7. For complex structures, use Node.js script for custom parsing
8. If Archive.org rate-limits, use a different mirror or wait

** ALWAYS Verify Conversion (CRITICAL)

After converting, ALWAYS check the output:

: head -20 "[file].org"     # Check header/title/author lines
: wc -l "[file].org"      # Should have substantial content
: tail -10 "[file].org"   # Check end is not truncated

If conversion failed:
- Output shows HTML 404/redirect errors
- File is tiny (<1KB) or empty
- Output contains =<html>= tags instead of text

FIX: Delete the file and try a different source:
- On Gutenberg: try =.txt.utf-8= instead of =[id].txt=
- On Archive.org: try =_djvu.txt= instead of =_text.txt=
- Try a different source entirely

** Archive.org Rate Limiting Workarounds

- Use CDN URLs directly: =dn000000.ca.archive.org=
- Wait between requests if 403/429 errors
- Try different item IDs from search results
- Don't spam downloads — space them out

* IMPORTANT: Output Format

=ALWAYS= convert downloaded books to =.org= format for Emacs.

Never keep =.txt=, =.html=, =.pdf= as the primary format.
The =.org= file is the final deliverable.

Why .org?
- Native Emacs format with outline structure
- Headings, folding, and navigation
- Annotations, org-capture, and org-agenda integration
- Full-text search within Emacs
- Portable, plain-text, and future-proof
